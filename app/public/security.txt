## Responsible Testing Policy

Testing the application logic of this site is explicitly permitted, provided that any findings are responsibly disclosed to the contact email above.

Please note: this email address may change once my custom domain email is fully configured.

---

## Scope

### In Scope

- Application logic
- Markdown rendering and content processing
- Authentication and authorization issues
- Admin dashboard and analytics endpoints (e.g. `/admin/login`)
- Headers, DNS issues, etc.

### Out of Scope

- The underlying server or host system
- Network infrastructure
- Third-party services or external APIs
- If vulnerability is found all I ask is no illicit access.

---

## Rules of Engagement

You are welcome to manually test both the site and its logic. That said:

- Do not spam HTTP/S traffic
- Do not run large automated scanners
- Do not be disruptive

Rate limiting is enforced via Flask-Limiter, and abusive behavior will result in IP bans via fail2ban or equivalent controls.
If you rely on rotating proxies to hammer a static blog, that is not research. That is a lifestyle choice.

If you want to test creatively and responsibly, great.
If you want to spray tools and hope something sticks, this is not the place.

---

## Disclosure Expectations

- Clear explanation of the issue
- Reproducible steps
- Impact assessment
- Suggested remediation (if available)
